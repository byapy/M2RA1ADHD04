TypeScript

¿Qué es TypeScript?

TypeScript es un lenguaje de programación de código abierto desarrollado y mantenido por Microsoft que amplía las capacidades de JavaScript mediante la incorporación de un sistema de tipado estático, herramientas avanzadas de desarrollo y características orientadas a la programación a gran escala.

TypeScript no reemplaza a JavaScript; en realidad, está construido sobre él. Todo código JavaScript válido también es válido en TypeScript. La principal diferencia es que TypeScript añade una capa de análisis y verificación que permite detectar errores durante el desarrollo antes de ejecutar el programa.

¿Qué ventajas ofrece TypeScript frente a JavaScript puro?

La ventaja más importante de TypeScript es su sistema de tipado estático. En JavaScript, los errores relacionados con tipos de datos normalmente se descubren durante la ejecución del programa. En cambio, TypeScript detecta muchos de estos errores antes de ejecutar el código, durante la compilación.

TypeScript obliga al desarrollador a definir estructuras más claras y controladas. Esto disminuye errores de lógica, uso incorrecto de variables e incompatibilidades de datos. Gracias a ello, el código es más predecible y robusto.

En proyectos grandes, TypeScript mejora el mantenimiento porque documenta automáticamente tipos, hace más comprensible el propósito de variables y funciones, y facilita localizar errores y modificar funcionalidades.

¿Cómo se instala TypeScript?

Para instalar TypeScript es necesario contar previamente con Node.js, ya que TypeScript se distribuye mediante npm, el administrador de paquetes incluido con Node.js.

Una vez instalado Node.js, se debe verificar que tanto Node.js como npm funcionen correctamente ejecutando los comandos node -v y npm -v.

La instalación de TypeScript se realiza con el comando npm install -g typescript. Después se recomienda verificar la instalación con tsc -v.

¿Cómo se compila un archivo TypeScript para convertirlo a JavaScript?

Este proceso es necesario porque los navegadores no ejecutan TypeScript directamente, sino JavaScript.

La compilación se realiza mediante el compilador oficial de TypeScript, conocido como tsc. Para convertir un archivo se usa la terminal con un comando como:

tsc nombreArchivo.ts

¿Qué extensión de archivo se utiliza para los archivos TypeScript?

Los archivos TypeScript usan la extensión .ts.

¿Cómo se define un tipo de dato en TypeScript?

En TypeScript los tipos de datos se definen mediante anotaciones de tipo, las cuales indican qué clase de valor puede almacenar una variable, parámetro, función u objeto.

let nombre: string = "Carlos";
let edad: number = 25;
let activo: boolean = true;
function sumar(a: number, b: number): number {
    return a + b;
}
let numeros: number[] = [1, 2, 3, 4];
interface Persona {
    nombre: string;
    edad: number;
}

let usuario: Persona = {
    nombre: "Ana",
    edad: 30
};

También existe la inferencia de tipos. Por ejemplo, si se escribe let mensaje = "Hola";, TypeScript entiende que la variable es de tipo string.

¿Qué son los tipos de unión y cómo se declaran?

Los tipos de unión permiten que una variable, parámetro o retorno de función acepte más de un tipo de dato posible.

let id: number | string;

id = 10;
id = "ABC123";

El símbolo | permite combinar tipos. Esto aporta flexibilidad sin perder el control del tipado.

function imprimir(valor: string | number): void {
    console.log(valor);
}

¿Qué es un tipo literal en TypeScript y para qué se utiliza?

Un tipo literal permite restringir una variable a un valor específico y exacto, en lugar de permitir cualquier valor de un tipo general.

let estado: "activo";

estado = "activo";
let direccion: "izquierda" | "derecha";

direccion = "izquierda";

Esto es útil para controlar estados u opciones limitadas dentro de una aplicación.

¿Qué son los tipos enumerados (Enums) y cómo se crean?

Los enums permiten definir un conjunto de valores constantes con nombre. Ayudan a representar opciones limitadas y relacionadas dentro de un programa.

enum Color {
    Rojo,
    Verde,
    Azul
}

let c: Color = Color.Verde;
enum Estado {
    Activo = "ACTIVO",
    Inactivo = "INACTIVO",
    Pendiente = "PENDIENTE"
}

let estadoActual: Estado = Estado.Activo;

¿Cómo se define un tipo alias y cuándo es útil?

Un tipo alias crea un nombre personalizado para un tipo de dato. Sirve para reutilizar tipos complejos y mejorar la legibilidad del código.

type ID = number | string;

let usuarioId: ID;

usuarioId = 10;
usuarioId = "ABC123";
type Usuario = {
    nombre: string;
    edad: number;
    activo: boolean;
};

let persona: Usuario = {
    nombre: "Ana",
    edad: 30,
    activo: true
};

¿Cómo se definen las funciones con tipos en TypeScript?

Las funciones con tipos se definen indicando el tipo de los parámetros y el tipo de valor que la función devuelve.

function sumar(a: number, b: number): number {
    return a + b;
}
function mostrarMensaje(mensaje: string): void {
    console.log(mensaje);
}
function saludar(nombre: string = "Usuario"): string {
    return "Hola " + nombre;
}

¿Qué son los parámetros opcionales y los parámetros predeterminados en funciones?

Los parámetros opcionales y predeterminados permiten definir funciones más flexibles, sin obligar a enviar todos los valores en cada llamada.

Los parámetros opcionales se indican con ? después del nombre del parámetro.

function saludar(nombre?: string): string {
    return "Hola " + (nombre ?? "invitado");
}

Los parámetros predeterminados tienen un valor inicial que se usa cuando no se proporciona otro valor.

function crearUsuario(nombre: string = "Usuario"): string {
    return nombre;
}

¿Cuáles son algunas mejores prácticas para escribir código limpio y mantenible en TypeScript?

Escribir código limpio y mantenible en TypeScript significa estructurar el proyecto de forma clara, predecible y fácil de escalar, aprovechando el sistema de tipos del lenguaje.

Algunas buenas prácticas son usar tipos claros, evitar el abuso de any, separar responsabilidades, nombrar variables y funciones de forma descriptiva, y mantener funciones pequeñas y fáciles de probar.
